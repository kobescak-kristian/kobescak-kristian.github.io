How I work

02 / METHOD

Rules that are enforced by tooling, not remembered by a person. All three live in the repos, where you can check them.

Gates before code

Eval thresholds are committed before the first line of system code — results recorded pass or fail. Passing is not assumed; it is measured against a bar set in advance.

Standards that enforce themselves

Every repo validates its own documentation with a pre-push hook and carries at least one code-verified architecture decision record. The standard is in the tooling, not in good intentions.

Independent verification

A second model re-derives every claim from a fresh public clone before anything is called done. Self-report is never the final word.

FAILURE · ON RECORD
One engine's keyed run failed its pre-committed gate. It shipped with the failure analyzed in the open — not silently re-tuned until the number looked right.
